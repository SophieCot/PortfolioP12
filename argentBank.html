Argent Bank

Contexte et Objectifs

Contexte : Argent Bank est une nouvelle banque en ligne qui souhaite se démarquer dans le secteur bancaire en proposant une application moderne et optimisée. Dans ce cadre, l’équipe technique travaille sur la mise en place d’un tableau de bord utilisateur basé sur React et Redux. Objectifs : Phase 1 : Authentification des utilisateurs Développer l’application React complète et responsive. Implémenter un système d’authentification sécurisé (connexion, déconnexion). Restreindre l’accès aux informations personnelles aux utilisateurs connectés. Permettre à l’utilisateur de modifier son pseudo uniquement. Utiliser Redux pour la gestion du state. Optimiser le code et les ressources (Green Code). Phase 2 : Gestion des transactions Afficher les transactions du mois en cours (groupées par compte). Détail d’une transaction sur une vue dédiée. Ajouter, modifier ou supprimer des informations sur une transaction (mais pas la transaction elle-même). Proposer une modélisation des routes API pour ces fonctionnalités, en respectant la documentation Swagger.

Fonctionnalités Principales

    Authentification et Gestion du Profil:

  • Connexion sécurisée avec gestion des erreurs (mauvais identifiants).
  • Déconnexion propre avec réinitialisation du state Redux.
  • Accès restreint aux informations du profil après connexion.
  • Modification possible du pseudo uniquement (pas du prénom ni du nom).

    • Transactions:

  • Affichage des transactions mensuelles par compte.
  • Affichage détaillé d’une transaction spécifique.
  • Modification des informations associées à une transaction.
  • Respect des bonnes pratiques Green Code (optimisation des images et composants réutilisables).

Outils et Technologies

Les principaux outils et technologies utilisés dans ce projet sont :

  • React.js : Développement des interfaces et composants.
  • Redux : Gestion du state global de l’application.
  • React Router : Navigation entre les pages.
  • Swagger (YAML) : Documentation et conception des routes API.
  • Node.js / Express : Back-end et API REST (déjà existants).
  • GitHub : Suivi du projet et gestion des issues.
